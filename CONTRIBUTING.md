# Contributing Guide

Thank you for your interest in this project.

This repository represents the **personal founder portfolio** of **Ziyaaddin Yaramis**, showcasing real-world experience, projects, and product work under **ZyInnova**.

While this is not a community-driven open source project, feedback and small improvements are welcome under the guidelines below.

---

## 🧭 Scope of Contributions

This repository is **not accepting large feature additions or redesigns**.

### Appropriate contributions include:

- Bug fixes (layout, accessibility, minor JS issues)
- Typo or content corrections
- Performance or semantic HTML improvements
- Accessibility improvements (ARIA, keyboard navigation)
- Documentation fixes (README, comments)

### Out of scope:

- Major design changes
- Rebranding or content rewrites
- New sections or features without prior discussion
- Framework migrations (React, Vue, etc.)

---

## 🛠 How to Contribute

1. **Fork** the repository
2. Create a new branch:

```bash
git checkout -b fix/short-description
```

3. Make your changes
4. Keep commits small and clear
5. Open a Pull Request that explains:
- What was changed
- Why the change is useful

---

✅ Code Style Guidelines

Please follow the existing style and structure.

HTML
	•	Use semantic tags whenever possible
	•	Keep indentation clean and consistent
	•	Avoid unnecessary wrapper elements

CSS
	•	Use existing design tokens defined in :root
	•	Avoid introducing new colors or fonts
	•	Keep selectors simple and readable

JavaScript
	•	Vanilla JavaScript only
	•	No new dependencies
	•	Prefer clarity over clever or complex solutions

⸻

🧪 Testing

Before submitting a Pull Request:
	•	Test on desktop and mobile screen sizes
	•	Test keyboard navigation (Tab / Shift + Tab)
	•	Ensure no console errors are introduced

⸻

📝 Commit Message Format

Use short, descriptive commit messages:

```text
fix: correct contact section spacing
docs: update README deploy section
style: improve focus-visible outline
```

⸻

🤝 Review Process

All Pull Requests are reviewed by the repository owner.

Acceptance depends on:
- Alignment with the project’s purpose
- Code quality and clarity
- Minimal impact on overall design consistency

⸻

📬 Questions or Suggestions

If you have a question or suggestion, feel free to open an issue
or contact directly:

📧 contact@zyinnova.com

⸻

Thank you for respecting the purpose and direction of this project.
